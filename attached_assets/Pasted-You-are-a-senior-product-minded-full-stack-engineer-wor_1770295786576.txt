B) BINANCE API DATA MODEL (PREFILLED + LOCKED)
========================================================
Use Binance REST endpoints only.

For spot market (default):
- Exchange info: GET /api/v3/exchangeInfo (symbols, base/quote, status, filters)
- 24h stats: GET /api/v3/ticker/24hr?symbol=BTCUSDT
- Price: GET /api/v3/ticker/price?symbol=BTCUSDT (optional if 24hr includes lastPrice)
- Order book: GET /api/v3/depth?symbol=BTCUSDT&limit=10
- Recent trades: GET /api/v3/trades?symbol=BTCUSDT&limit=20
- Klines: GET /api/v3/klines?symbol=BTCUSDT&interval=1h&limit=500

Define a canonical locked object in our system: BinancePrefill
(Everything read-only in CMS)

BinancePrefill.lockedIdentity:
- binanceSymbol (e.g., SOLUSDT) [primary identifier]
- baseAsset, quoteAsset
- status
- tickSize, stepSize, minNotional (from exchangeInfo filters)
- fetchedAt

BinancePrefill.lockedMarketSummary:
- lastPrice
- priceChange
- priceChangePercent
- openPrice
- highPrice
- lowPrice
- volume (base)
- quoteVolume
- bidPrice, bidQty
- askPrice, askQty
- weightedAvgPrice
- count (trades)
- closeTime (asOf)
- computedSpreadPct = (ask-bid)/mid * 100
- fetchedAt

BinancePrefill.lockedChart:
- interval
- series[] (t, open, high, low, close, volume)
- fetchedAt

BinancePrefill.lockedOrderBook:
- bids[] (price, qty)
- asks[] (price, qty)
- lastUpdateId
- fetchedAt

BinancePrefill.lockedTrades:
- trades[] (price, qty, time, isBuyerMaker)
- fetchedAt

All fields are predefined and can be null if the endpoint fails; show “Unavailable”.
